# リポジトリ責務定義

## 概要

本ドキュメントは、FRONTLINE LOGIC - 野戦乃庭 - における
YasenSim と FLL のリポジトリ責務を定義する。

## YasenSim

YasenSim は、FRONTLINE LOGIC - 野戦乃庭 - の
研究開発およびプレイ可能プロトタイプ用リポジトリである。

### 責務

- 大量シミュレーションデータの集積
- 分析基盤の整備
- バランス調整基盤の整備
- AI 挙動の検証
- Commander Mode v0-v2 の開発
- ゲーム要件を満たす playable prototype の構築

### 定義

YasenSim における「ゲーム要件」とは、
ゲーム性をプレイできることを指す。

YasenSim は単なる研究用サンドボックスではない。
Commander Mode v0-v2 において、
ゲームとしてプレイ可能な状態まで到達することを目的とする。

ただし、YasenSim は最終製品リリース用リポジトリではない。

## FLL

FLL は、FRONTLINE LOGIC - 野戦乃庭 - の
プロダクト開発リポジトリである。

### 責務

- YasenSim からのソースコード、設計判断、バランス構造の引き継ぎ
- 検証済みゲームバランス構造の再現
- Commander Mode v3 相当の開発
- プロダクトリリースを目的とした実装

### 定義

FLL は製品化ラインである。
YasenSim で検証されたゲーム構造を、
リリース可能なゲームとして再構築することを目的とする。

## 開発フロー

### YasenSim

- 研究
- シミュレーション
- バランス検証
- AI 挙動検証
- Commander Mode v0-v2 のプレイ可能化

### FLL

- プロダクト実装
- リリース前提の開発
- Commander Mode v3 相当の構築

## まとめ

YasenSim は、ゲームがプレイ可能であり、
バランス構造として成立することを証明する。

FLL は、その証明済み構造を引き継ぎ、
製品として成立するゲームへ変換する。
